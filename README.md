# 本产品将代替你
1. 批量跑问答数据的繁琐
2. 用肉眼逐条审查几百条模型回答的枯燥劳动
  - 它将非结构化的 AI 对话，转化为可量化、可监控的质量评分表

# 您只需要做三件事：
1. 传 Excel ：上传包含“参考资料”列 和“评测集”列的表格。
2. 填写Prompt ：填写您用于回答评测集的prompt
3. 点击按钮

# 您将直接得到： 
1. 基于您prompt而生成的答案
2. 用来评测答案是否合您心意的“评测prompt”（如果不合心意可以手动修改）
3. 基于“评测prompt”+您数据源（知识库内容）+评测集+第一步生成的答案而得到的评测集评分

# 1. 环境准备与启动
## 1.1 前置要求
- 操作系统：Windows 10/11
- 环境：已安装 Python 3.8+
- 网络：能访问 OpenRouter 或 OpenAI API 的网络环境
  - 我自己接的是openrouter的api

## 1.2 启动工具
1. 进入工具目录，双击运行 启动器.bat
2. 浏览器会自动打开，访问地址通常为 http://localhost:8501。

# 2. 界面功能详解
[图片]
## 2.1 左侧设置栏
- API Base URL：
  - 如果使用的是openrouter的API，保持默认的即可https://openrouter.ai/api/v1
  - 如果您使用本地 Ollama，可改为 http://localhost:11434/v1。
- API Key：
  - 在此输入您的API KEY
- Model Name：
  - 输入您想调用的模型名称（例如 google/gemini-2.5-flash 或 gpt-4o）
  - 注意：本产品无需联网，密钥仅在本地内存运行，关闭程序后即清除，如果点击保存配置也是保存在您本地（保存后重开网页就无需重新输入了）。
## 2.2 数据导入区
- 上传文件：点击“Browse files”上传 .xlsx 格式的 Excel 文件。
- 列名映射：
  - 系统会自动尝试匹配列名。
    - 数据源列：请在此列输入您评测集的知识库知识内容
    - 问题集列：请在此列输入您的评测集
    - 生成答案列：此列将由AI回答您评测集的答案
    - 评测结果列：此列将由AI评测您评测集的答案

# 3. 操作流程 

## 3.1 批量生成答案
1. 编辑 System Prompt：
  - 在文本框中输入指令，例如：“你是一个客服助手，你的任务是……”
2. 点击“执行生成”：
  - 程序会逐行调用您API并遵循您的system prompt生成答案。
  - 进度条会实时显示处理进度。
    
## 3.2 智能生成“评测prompt”
1. 点击“自动生成评测 Prompt”：
  - 系统会分析您的system prompt
  - 生成一套包含“准确性”、“完整性”等维度的评分标准。
  - 关键特性：生成prompt要求模型只输出评测答案，方便后续统计。
2. 确认 Prompt：
  - 您可以手动修改生成的 Prompt。
  - 确认无误后，点击 “确认 Prompt 并继续”。
    
## 3.3 批量自动打分
此步骤让 AI 扮演“裁判员”，参照您的知识来源，判断评测结果是否正确
1. 点击“开始评测 ”：
  - 程序会再次遍历所有数据，将“数据源+评测集+模型回答”发送给裁判模型。
  - 裁判模型返回 0-5 的分数。

# 4. 结果导出

- 处理完成后，页面底部会出现 “下载评测结果 (Excel)” 按钮。
- 点击下载，您将获得一份包含原始数据、模型回答、最终评分的完整 Excel 报表。
  
# 5. 常见问题

- 如果任何按钮按不了，请检查您的API Base URL、API Key、Model Name是否正确
- 如果您觉得在本产品里输入“System Prompt”不能调用知识库，您可以将您智能体的API接过来，并在本产品的“System Prompt”中填"/"，一样可以达到一键生成的效果
- 如果您按照上一条规则操作，后续操作可以填写一份新的API KEY，并把您的“System Prompt”补上，后续内容一样可以接着执行
